# 要件定義書 — X Image Catcher

## 1. 背景・課題

X（旧Twitter）のタイムラインでは投稿が次々と流れていく。
気になる画像を見かけても、いいね・ブックマークをする前にスクロールで見失ってしまい、
後から探し出すことが困難である。

---

## 2. 目的

Xをブラウジング中に表示された画像を自動で記録し、
後から落ち着いてレビュー・いいね・ブックマークできる環境を提供する。

---

## 3. スコープ

| 項目 | 内容 |
|------|------|
| 対象プラットフォーム | x.com（旧 twitter.com） |
| 対象ブラウザ | Google Chrome（Manifest V3） |
| 対象ページ | タイムライン（ホーム・検索・プロフィール等） |
| 保存形式 | 画像URL + ツイートメタデータ（ローカル保存） |

---

## 4. 機能要件

### 4.1 自動キャプチャ（content script）

| ID | 要件 |
|----|------|
| FR-01 | x.com を開いているとき、タイムラインに画像が表示されると自動でキャプチャする |
| FR-02 | キャプチャ対象は `pbs.twimg.com/media/` ドメインの画像とする |
| FR-03 | 同一画像（同一ツイート内）は重複して保存しない |
| FR-04 | 保存するデータは以下の通りとする |

**保存データ構造**

```json
{
  "id": "tweet_<tweetId>_<imageIndex>",
  "tweetId":     "ツイートID",
  "tweetUrl":    "https://x.com/<handle>/status/<tweetId>",
  "authorName":  "表示名",
  "authorHandle":"@handle",
  "authorAvatar":"https://pbs.twimg.com/profile_images/...",
  "imageUrl":    "https://pbs.twimg.com/media/...?format=jpg&name=large",
  "capturedAt":  "ISO 8601 timestamp"
}
```

### 4.2 ギャラリー表示（popup）

| ID | 要件 |
|----|------|
| FR-05 | 拡張機能アイコンをクリックするとポップアップでギャラリーが開く |
| FR-06 | 保存済み画像をグリッドレイアウトで新しい順に表示する |
| FR-07 | 画像をクリックすると、元のツイートを新しいタブで開く |
| FR-08 | 画像ごとに「削除」ボタンを表示し、個別削除できる |
| FR-09 | 「すべて削除」ボタンで全件削除できる |
| FR-10 | 未確認（新規キャプチャ）件数を拡張アイコンのバッジに表示する |
| FR-11 | 投稿者のアバター・ハンドル名をサムネイル上に表示する |

---

## 5. 非機能要件

| ID | 要件 |
|----|------|
| NFR-01 | 画像ファイル本体はローカルに保存しない（URL参照のみ） |
| NFR-02 | chrome.storage.local を使用し、外部サーバーへの通信は行わない |
| NFR-03 | ストレージ上限（chrome.storage.local は 10MB）を考慮し、保存件数は最大 1,000 件とする。超過時は古い順に削除する |
| NFR-04 | キャプチャ処理はページのパフォーマンスに影響を与えないよう非同期で実行する |
| NFR-05 | ポップアップは幅 400px × 高さ 600px 程度を基準とする |

```
┌────────────────────────────────┐
│ 📷 X Image Catcher   [全削除]  │  ← ヘッダー
│ 123件保存済み                  │
├────────────────────────────────┤
│ [img][img][img]                │
│ [img][img][img]  ← グリッド   │
│ [img][img][img]                │
│  ...（スクロール可）            │
└────────────────────────────────┘

各サムネイル:
┌─────────────┐
│             │
│   画像      │  ← クリックでツイートを開く
│             │
│ @handle  [✕]│  ← ハンドル + 削除ボタン
└─────────────┘
```

## 8. 技術スタック

| 項目 | 採用技術 |
|------|---------|
| 拡張形式 | Chrome Extension Manifest V3 |
| キャプチャ | MutationObserver + IntersectionObserver |
| ストレージ | chrome.storage.local |
| フロントエンド | HTML / CSS / Vanilla JS（ビルドなし） |
